# MVP-0001: Generalized 3D Reconstruction Pipeline (Tomographic reconstruction)

Status: Draft
Version: 0.1
Author: Aydrian Plotnikov
Target: Plasma DBD Plane Physical Model
Date: April 19th, 2026

## 1. Abstract

Develop a system that constructs a 3D volumetric representation fro 2D input data, with a design that:

- Supports multiple input implementations:
  - Pre-reconstructed 2D slices
  - Raw line-integral projection data
- Implements Filtered Backprojection (FBP)
- Produces measurable outputs for:
  - performance benchmarking
  - reconstruction quality
- Remains geometry-agnostic, enabling future support for:
  - parallel-beam
  - fan-beam
  - cone-beam systems

This extensibility allows for making new implementations as idea's grow.

## 2. Scope

### Included

- Ingestion of 2D data (slice-based or projection-based)
- Construction of 3D volume
- Implementation of parallel-beam FBP, slice-wise
- Visualization of results--2D slices and 3D representation
- Benchmarking of runtime and resource usage
- Validation against known or reference data

### Excluded

- Hardware-specific optimizations
- Full support for fan-beam or cone-beam reconstruction
- Iterative reconstruction methods
- Extended physics modeling

## 3. Input Modalities

The system must support two conceptual input types:

### 3.1 Reconstructed Slice Data

- A sequence of 2D images representing cross-sectional slices
- Assumed to already encode spatial information
- Used to validate volume construction

### 3.2 Projection Data

- Measurements representing integrals of a field along rays
- Organized as:
  - Sinograms (parallel-beam)
  - or equivalent projection stacks
- Used as input for reconstruction algorithms

## 4. Output

The system produces:

- A 3D voxel volume
- Derived views:
  - axial / sagittal / coronal slices
  - 3D surface/volume rendering

## 5. Reconstruction Approach

Baseline Algorithm: Filtered Backprojection (FBP)

- Implemented in parallel-beam geometry
- Applied slice-wise:
  - each slice reconstructed independently from its corresponding projection data
- Chosen because:
  - computationally efficient (more so then iterative reconstruction)
  - analytically well understood
  - widely used and implemented

## 6. Data Strategy

### 6.1 Synthetic Data

Used for algorithm validation.

### 6.2 Real Data

Candidate Dataset:

- [LDCT-and-Projection-data](https://www.cancerimagingarchive.net/collection/ldct-and-projection-data/)

Rationale:

- contains both projection data and constructed slices
- listed specifically to validate tomographic reconstruction models

## 7. Benchmarking Requirements

### 7.1 Performance

- Total reconstruction time
- Time per slice
- Time per volume
- Scaling behavior with image resolution and number of projection angles

### 7.2 Quality

For synthetic data:

- RMSE
- PSNR

For real data:

- visual comparision against reference slices

### 7.3 Resource Usage

- Memory footprint
- Estimated data movement
- Identification of dominant cost

## 8. Summary

This MVP establishes a geometry-agnostic reconstruction pipeline that:

- supports both slice-based and projection-based inputs
- implements a baseline FBP algorithm
- produces measurable performance and quality metrics
- enables informed decisions about:
  - reconstruction methods
  - hardware targets
  - system architecture
